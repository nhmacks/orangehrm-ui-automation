## Key Design Patterns

### 1. **Singleton Pattern**
- `BrowserManager` - Single browser instance per worker
- `ConfigManager` - Single config instance

### 2. **Page Object Model (POM)**
- Encapsulation of UI elements and actions
- Reusability across tests
- Maintainability

### 3. **Factory Pattern**
- Page creation through constructors
- Browser/context creation

### 4. **Strategy Pattern**
- Different configs per environment
- Different execution profiles

### 5. **Observer Pattern**
- Hooks observe test lifecycle
- Logger observes all actions
